## 1. 产品概述

### 1.1 产品定位

**产品名称**：待定（AI Mail）  
**产品类型**：AI 驱动的 iOS 邮箱客户端  
**目标用户**：日常生活需要管理多个邮箱的普通消费者  
**核心价值**：用 AI 简化邮件处理流程，让用户少花时间在邮件上

### 1.2 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 多邮箱聚合 | 支持 IMAP/SMTP 协议，绑定所有标准邮箱 | P0 |
| AI 邮件摘要 | 自动生成每封邮件的核心内容摘要 | P0 |
| AI 分类处理 | 自动分类：重要/普通/促销/社交，退订管理 | P0 |
| AI 起草回复 | 一键生成回复草稿，用户修改后发送 | P0 |
| 日程提取 | 自动识别邮件中的日程信息，添加到日历 | P1 |
| 模糊语义搜索 | 支持自然语言搜索，如"和老板讨论的预算" | P0 |
| 关键词搜索 | 传统关键词、短语、日期搜索 | P0 |
| Hybrid 搜索 | 结合语义和关键词的混合搜索 | P0 |

### 2.2 AI Pipeline

#### 邮件处理流程

```
新邮件到达
    │
    ▼
┌───────────────────────────────────────┐
│  Step 1: 本地预处理（Core ML）         │
│  - 垃圾邮件分类                        │
│  - 优先级判断（重要/普通/促销）         │
│  - 自动打标签（工作/个人/订单）         │
│  耗时：< 100ms                         │
└───────────────────────────────────────┘
    │
    ▼
┌───────────────────────────────────────┐
│  Step 2: 本地语义索引（FAISS）          │
│  - 生成邮件内容向量                     │
│  - 存入本地向量数据库                  │
│  - 用于模糊语义搜索                     │
└───────────────────────────────────────┘
    │
    ▼
┌───────────────────────────────────────┐
│  Step 3: 云端深度处理（GPT-4o mini）    │
│  - 邮件摘要生成（英文/中文）            │
│  - Action Items 提取                  │
│  - 日程信息识别 → 日历事件             │
│  耗时：500-2000ms                     │
└───────────────────────────────────────┘
    │
    ▼
┌───────────────────────────────────────┐
│  Step 4: 用户可见呈现                   │
│  - 收件箱分类展示                      │
│  - 邮件摘要预览                        │
│  - AI 回复建议按钮                     │
└───────────────────────────────────────┘
```

#### 搜索流程（Hybrid Search）

```
用户输入查询："和老板讨论的预算"
    │
    ▼
┌───────────────────────────────────────┐
│  关键词搜索（SQLite FTS5）             │
│  匹配："老板" "预算"                  │
│  → 候选结果 A                         │
└───────────────────────────────────────┘
    │
    ▼
┌───────────────────────────────────────┐
│  语义搜索（FAISS）                     │
│  查询向量匹配相关语义                   │
│  → 候选结果 B                         │
└───────────────────────────────────────┘
    │
    ▼
┌───────────────────────────────────────┐
│  结果合并 + 重排序                      │
│  - 关键词命中权重：40%                 │
│  - 语义相似度权重：60%                 │
│  → 最终排序结果                         │
└───────────────────────────────────────┘
```

## 5. 功能优先级

### Phase 1 — MVP（P0 核心）

| 功能 | 描述 |
|------|------|
| 邮箱绑定 | IMAP/SMTP 绑定单个或多个邮箱 |
| 邮件列表 | 展示收件箱邮件，含发件人、主题、摘要预览 |
| 邮件详情 | 查看完整邮件内容 |
| AI 摘要 | 对单封邮件生成一句话摘要 |
| 关键词搜索 | SQLite FTS5 全文搜索 |
| 语义搜索 | FAISS 向量搜索 |
| Hybrid 搜索 | 两者结合，合并排序 |
| AI 起草回复 | 生成回复草稿 |

### Phase 2 — 完善（P1）

| 功能 | 描述 |
|------|------|
| AI 分类 | 自动将邮件分为：重要/普通/促销/社交 |
| 退订管理 | 识别退订链接，一键退订 |
| 日程提取 | 从邮件中提取日程，创建日历事件 |
| Action Items | 提取邮件中的待办事项 |

### Phase 3 — 增强（P2）

| 功能 | 描述 |
|------|------|
| 多语言摘要 | 支持中英文邮件摘要 |
| 批量处理 | 选中多封邮件，批量 AI 处理 |
| 邮件模板 | 保存常用回复模板 |

## 6. 数据流

### 6.1 邮件同步

```
用户添加邮箱账号
    │
    ▼
IMAP 凭证安全存储（Keychain）
    │
    ▼
MailCore2 连接邮箱服务器
    │
    ▼
抓取邮件头（From/Subject/Date）→ 展示列表
    │
    ▼
按需拉取邮件正文（懒加载）
    │
    ▼
存储到本地 SQLite 数据库
    │
    ▼
同时触发 AI 处理流程
```

### 6.2 AI 处理

```
邮件入库
    │
    ▼
Core ML 分类（< 100ms，本地）
    │
    ├── 是垃圾邮件 → 标记，归档
    ├── 促销邮件 → 标记，归档
    └── 正常邮件 → 继续
    │
    ▼
生成 Embedding → 存入 FAISS
    │
    ▼
发送云端 API 请求（GPT-4o mini）
    │
    ├── 摘要
    ├── Action Items
    └── 日程信息
    │
    ▼
存储结果，更新 UI
```

## 7. 安全与隐私

| 方面 | 实现 |
|------|------|
| 凭证存储 | iOS Keychain，生物识别解锁 |
| 传输加密 | IMAP over TLS / SSL |
| 本地数据 | SQLite 数据库，本地加密 |
| 云端 AI | 仅发送必要内容，不存储邮件内容 |
| 隐私政策 | 不收集、不分享用户邮件数据 |

## 8. 非功能性要求

| 指标 | 目标 |
|------|------|
| 冷启动时间 | < 2 秒 |
| 邮件列表滚动 | 60 FPS |
| AI 摘要延迟 | < 3 秒（云端） |
| 搜索响应时间 | < 500ms |
| 电池消耗 | 后台低功耗同步 |
| 离线支持 | 查看已同步邮件，AI 功能需联网 |

## 10. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| MailCore2 维护状态 | 库停止更新 | 考虑切换到 Custom IMAP 实现 |
| 云端 AI 成本 | 按 token 计费 | 本地处理简单任务，限制 API 调用 |
| 邮件隐私 | 用户敏感数据 | 明确隐私政策，最小化数据发送 |
| iOS 推送通知 | 实时性要求 | 本地通知 + 后台刷新 |
| 中文分词 | FTS5 中文支持 | 使用 ICU 中文分词器 |
